So läuft's

Drei Schritte vom „Hi" zum gemergten PR

Kein bürokratisches CLA, keine 50-seitige Contributor-Doku. So einfach geht's:

01

Hallo sagen

Im GitHub Discussion-Thread oder per E-Mail. Sag wer du bist und woran du Interesse hast, wir antworten zeitnah.

Reaktionszeit ≤ 24h
02

Issue wählen oder vorschlagen

Such dir was aus den offenen Issues, oder erstell ein neues. Wir kommentieren mit Kontext und Hinweisen, wo du im Code anfängst.

Mentoring inklusive
03

Beitrag einreichen

Pull Request, Übersetzung, Doku-Patch oder Markdown im Issue — alles passt. CI rennt, wir reviewen, dann mergen wir.

Code of Conduct

Wie wir hier miteinander umgehen

Mandari folgt dem Contributor Covenant 2.1. Kurz gesagt:

  • Respekt zuerst. Keine Beleidigungen, keine Diskriminierung, egal wie hitzig die Diskussion wird.
  • Konstruktive Kritik. An der Sache, nicht an der Person.
  • Geduld mit Anfänger:innen. Jede:r war mal an Tag eins.
  • Privatsphäre. Keine Daten ohne Zustimmung weitergeben.

Verstöße bitte direkt an conduct@mandari.de, vertraulich.
